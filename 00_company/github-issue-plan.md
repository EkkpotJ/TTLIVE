# GitHub Issue Plan

ระบบติดตามงานของ TTLIVE ควรใช้ Issue หลัก 1 อันสำหรับภาพรวม และใช้ comment เป็น update log

## Main Issue

ชื่อ issue แนะนำ:

```text
TTLIVE Operating System Setup
```

## Main Checklist

```md
- [ ] Channel profile
- [ ] Games and streaming categories
- [ ] Streaming availability
- [ ] Brand guideline
- [ ] Roles and permissions
- [ ] Live operation workflow
- [ ] Schedule system
- [ ] Asset and scene system
- [ ] Clip pipeline
- [ ] Analytics and reports
- [ ] Automation
- [ ] Community
- [ ] Monetization
```

## Comment Format

ใช้ format นี้ทุกครั้งที่มี update สำคัญ:

```md
## Update: YYYY-MM-DD

### Summary
- ...

### Changed Files
- ...

### Commit
`commit-hash`

### Next
- ...
```

## When To Create Child Issues

ยังไม่ต้องแตก issue ย่อยจนกว่างานใหญ่และชัด เช่น:

- Build full overlay pack
- Create automated clip workflow
- Set up real team permissions
- Design sponsor package
- Build dashboard app

## Current Rule

ตอนนี้ใช้ issue หลักเป็นศูนย์กลางก่อน เพื่อไม่ให้การอัปเดตกระจายและสับสน
